React components as jQuery plugins

React is awesome. You've seen it, you've tried it, you've loved it. Fast rendering, a lack confusing two-way-binding, a built-in disdain for architecture astronautics, and very little boilerplate. Everything you've ever wanted in a web framework.

At the very least, React looks like a promising new toy.

But unless you're a Facebook or something, fun new toys don't print money. Wordpress and Drupal and the PHP of old glued together with many jQuery plugins print money. Users don't care whether you're having fun with your code or not, they want results. They want to make money and feel like a million bucks and do the stuff they couldn't do before they hired you.

And so you look at React and you think "Man, that is going to be really awesome to use on the next project!".

Then the next project comes and it's got a tight deadline and you make it with the same old tech again because it's reliable and because it prints money and it's not that bad.

Fear not! You can turn React components into plain old jQuery plugins. That's right, you get to have your cake and eat it too.

You can learn the details on my blog, here. Or keep reading.

Real world interface

This is how the real world can use the above <Counter /> component:


$(".container .counter").clickCounter();

$(".btn-10x").click(function () {
    $(".container .counter")
        .clickCounter()[0]
        .val(10);
});

$(".btn-get").click(function () {
    var val = $(".container .counter")
            .clickCounter()[0]
            .val();

    alert("Current counter value is: "+val);
});

Pretty normal huh?

But you can use it like a normal React component, if you want:


const MyThing = React.createClass({
    render: function () {
        return (
            <div>
                <Counter />
            </div>
        )
    }
});

And if all else fails, you have the function options:


var RenderCounter = require('/path/to/counter.js');
RenderCounter(".container .counter");

// OR

window.RenderCounter(".container .counter");
